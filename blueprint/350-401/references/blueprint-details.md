**1.0 Architecture (15%)**
*   **1.1 Enterprise Network Design Principles:** **Hierarchical LAN design** (Core, Distribution, Access layers) provides fault containment and scalability compared to flat networks. You must understand the differences between two-tier (collapsed core) and three-tier designs. High availability is achieved by leveraging hardware redundancy, **First-Hop Redundancy Protocols (FHRPs)** like HSRP, VRRP, and GLBP, and Stateful Switchover (SSO) combined with Non-Stop Forwarding (NSF).
*   **1.2 Cisco Catalyst SD-WAN Solution:** The architecture includes the **vBond Orchestrator, vManage NMS, vSmart Controller, and SD-WAN Edge devices**. Deep dives require understanding **Cloud OnRamp for SaaS**, which uses HTTP probes and Viptela Quality of Experience (vQoE) scores to dynamically route traffic over the best-performing internet circuits.
*   **1.3 Cisco SD-Access Solution:** The SD-Access Underlay typically uses IS-IS for routed access, while the Overlay utilizes **VXLAN-GPO encapsulation** to carry Layer 2/3 traffic alongside Cisco TrustSec Scalable Group Tags (SGTs). You must understand fabric roles like Control Plane Nodes, Fabric Border Nodes, Fabric Edge Nodes, and Fabric WLCs. These are orchestrated through **Cisco DNA Center's Design, Policy, Provision, and Assurance workflows**.
*   **1.4 Quality of Service (QoS):** QoS models encompass Best Effort, IntServ (RSVP), and DiffServ. This domain requires mastering the **Modular QoS CLI (MQC)** with class-maps and policy-maps. Traffic classification uses Layer 2 (802.1p/CoS), Layer 3 (DSCP/IP Precedence), and **NBAR2** for Layer 7 deep packet inspection. Congestion management covers low-latency queuing (LLQ) and token bucket algorithms for policing and shaping (single-rate/two-rate three-color markers).

**2.0 Virtualization (10%)**
*   **2.1 Device Virtualization:** Understand server virtualization using Type 1 and Type 2 Hypervisors and Virtual Machines (VMs), contrasted with **Containers** which share a single operating system. Virtual Switching encompasses standard vSwitches and optimizations like **OVS-DPDK, PCI passthrough, and SR-IOV** to bypass kernel interrupts and improve throughput.
*   **2.2 Data Path Virtualization:** Includes configuring **Virtual Routing and Forwarding (VRF)** to isolate routing tables, and building overlay tunnels like Generic Routing Encapsulation (GRE) and IPsec. IPsec details require knowing ISAKMP/IKEv1/IKEv2 phases, AES/3DES encryption algorithms, and configuring Virtual Tunnel Interfaces (VTI) over IPsec.
*   **2.3 Network Virtualization:** Covers **Cisco Locator/ID Separation Protocol (LISP)** architecture, including Ingress/Egress Tunnel Routers (ITR/ETR) and Map Servers/Resolvers which map Endpoint Identifiers (EIDs) to Routing Locators (RLOCs). **VXLAN segments** use MAC-in-IP/UDP tunneling bounded by a VXLAN Network Identifier (VNI).

**3.0 Infrastructure (30%)**
*   **3.1 Layer 2:** Troubleshoot 802.1q trunking and VTP versions 1-3. Configure **EtherChannel** bundles using LACP and PAgP, manipulating port-channel minimum links and load-balancing hashes. Master STP and RSTP port states (Discarding, Learning, Forwarding) and enhancements like **Root Guard, BPDU Guard, BPDU Filter, Loop Guard, and UDLD**.
*   **3.2 Layer 3:** **OSPFv2/OSPFv3** fundamentals include LSA Types (1 Router, 2 Network, 3 Summary), DR/BDR elections on broadcast media, and configuring inter-area route summarization. **EIGRP** requires knowing the DUAL algorithm (Successor/Feasible Successor), classic vs. wide metric K-values, and unequal-cost load balancing via the variance multiplier. **BGP** routing demands a deep dive into best-path selection utilizing attributes like Weight, Local Preference, AS_Path, Origin, and MED.
*   **3.3 IP Services:** Interpret Network Time Protocol (NTP) stratums alongside **Precision Time Protocol (PTP)** boundary clocks. Configure NAT/PAT and FHRPs (HSRP, VRRP, GLBP), tying them to **object tracking** to decrement priorities upon line-protocol failures. Multicast protocols cover IGMPv2/v3, Protocol Independent Multicast (PIM) Dense/Sparse Modes, Rendezvous Points (RP), and Reverse Path Forwarding (RPF).

**4.0 Network Assurance (10%)**
*   **4.1 Diagnose Network Problems:** Utilize extended ping (modifying packet size, DF bit, ToS) and extended traceroute options. Pinpoint issues using **conditional debugging** bound to specific ACLs, and configure SNMP traps and syslog buffer severity levels.
*   **4.2 Flexible NetFlow:** Differentiate traditional NetFlow from Flexible NetFlow by configuring reusable components like **Flow Records (defining key and non-key fields)**, Flow Monitors, Flow Exporters, and Flow Samplers.
*   **4.3 SPAN/RSPAN/ERSPAN:** Implement Local SPAN sessions, Remote SPAN extending across trunked VLANs, and **Encapsulated Remote SPAN (ERSPAN)** which routes mirrored traffic over Layer 3 utilizing GRE encapsulation.
*   **4.4 IP SLA:** Configure continuous network probes monitoring delay, jitter, and packet loss utilizing **ICMP echo** and **HTTP GET** operations.
*   **4.5 Cisco Catalyst Center (DNA Center):** Operate the Assurance workflow which uses streaming telemetry and machine learning to build device health scores. Features like the **Client 360 view**, Network Time Travel, and the visual **Path Trace** tool help pinpoint physical paths and intermediate ACL drops.

**5.0 Security (20%)**
*   **5.1 Device Access Control:** Secure local lines (vty, con) utilizing usernames, SSHv2, and Privilege Levels (Role-Based Access Control). Implement AAA frameworks using TACACS+ and RADIUS servers, and understand 802.1X Extensible Authentication Protocol (EAP) flows and **MAC Authentication Bypass (MAB)**.
*   **5.2 Infrastructure Security:** Configure Zone-Based Firewalls (ZBFW) for stateful packet inspection leveraging `inspect` class-maps and policy-maps. Apply **Control Plane Policing (CoPP)** via MQC to safeguard the route processor using distinct policing rates and conform/exceed/violate actions.
*   **5.3 REST API Security:** Establish secure API interactions using HTTP Basic Authentication, token-based authentication (e.g., X-Auth-Token), and encryption.
*   **5.4 Network Security Design:** Familiarize with the **Cisco SAFE framework** architectural PINs (Campus, WAN, Edge, Cloud) and Cisco Talos threat intelligence. Implement Next-Generation IPS (Cisco Secure IPS) for Application Visibility and Control (AVC) and URL filtering. Leverage **Cisco TrustSec** for software-defined segmentation using Scalable Group Tags (SGT) alongside Layer 2 MACsec (802.1AE) encryption.

**6.0 Automation and Artificial Intelligence (15%)**
*   **6.1 & 6.2 Python and JSON:** Interpret Python scripts that rely on dictionaries (key/value pairs identical to JSON formats) and the `requests` module to programmatically interact with REST APIs.
*   **6.3 YANG:** Understand the standardized data modeling language defined in RFC 6020 which utilizes hierarchical tree structures composed of **containers, lists, and leaves** to separate configuration data from state data.
*   **6.4 & 6.5 Network APIs:** Interface with Cisco DNA Center and vManage using HTTP CRUD methods (POST, GET, PUT, DELETE) and accurately interpret standard HTTP response status codes like 200 (OK), 201 (Created), 401 (Unauthorized), and 403 (Forbidden).
*   **6.6 Embedded Event Manager (EEM):** Construct EEM applets utilizing Tcl scripts to automate on-box tasks using **event detectors** (matching syslog messages or CLI patterns) followed by **actions** (modifying configurations or sending emails).
*   **6.7 Orchestration Tools:** Contrast the architectures of agent-based tools like **Puppet** (using manifests and modules), **Chef** (using recipes and cookbooks), and **SaltStack** (using pillars and minions) against agentless automation tools like **Ansible**, which pushes YAML-based playbooks to hosts via SSH.
